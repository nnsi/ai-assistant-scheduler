# モバイルアプリUI改善タスク

## 概要
モバイルアプリ（http://localhost:8081/）のUIをよりスマホアプリらしく改善する。
現状はPC向けレイアウトを流用しており、モバイルに最適化されていない。

## UI問題点一覧

### 1. ヘッダー関連
- [ ] **ヘッダーが分離しすぎ**: アプリヘッダー（AI Scheduler + プロフィール）とカレンダーヘッダー（日付ナビ + ツールバー）が分離しており、縦に場所を取りすぎ
- [ ] **ヘッダーの階層が深い**: スマホアプリでは通常、ナビゲーションバーは1段で収める

### 2. ツールバー/ナビゲーション関連
- [ ] **フィルターボタンが多すぎ**: 検索、カテゴリ、カレンダー、条件の4つのボタンが常時表示されている
- [ ] **アイコン+テキストで冗長**: 各ボタンにアイコンとテキスト両方があり、スペースを浪費
- [ ] **月/週/日切替の見た目がPC的**: セグメントコントロールとしては良いが、配置がツールバー内で窮屈

### 3. カレンダービュー関連
- [ ] **月表示のセルが小さい**: 予定タイトルが切れて見えにくい
- [ ] **週表示が横に狭い**: 7日分を横に並べるため各列が狭く、予定が読みにくい
- [ ] **日表示のヘッダーが冗長**: 「2026年1月13日」と「火曜日」を別行で表示しており縦スペースを浪費

### 4. モーダル/ダイアログ関連
- [ ] **モーダルがフルスクリーンでない**: pageSheetスタイルで上部に隙間がある
- [ ] **予定詳細の情報が少ない**: 詳細画面に余白が多く、情報密度が低い
- [ ] **プロフィール設定の余白が多い**: 画面下半分がほぼ空白

### 5. フォーム関連
- [ ] **予定作成フォームの入力フィールド間隔が広い**: スクロールが多くなる
- [ ] **AIで補完ボタンが目立たない**: 薄いグレーで機能が分かりにくい

### 6. FAB・タップターゲット関連
- [ ] **FABの位置は適切だがサイズ最適化**: 56pxは標準的だが、視認性向上の余地あり
- [ ] **一部ボタンのタップ領域が小さい**: 予定カードのクリック領域など

### 7. 全体的な問題
- [ ] **PCレイアウトの流用**: スマホ専用のUIパターンが使われていない
- [ ] **余白の不統一**: 画面によって余白のサイズがバラバラ
- [ ] **視覚的階層が弱い**: どこを見るべきか一目で分かりにくい

## 修正方針

### Phase 1: ヘッダー統合とツールバー改善
1. アプリヘッダーとカレンダーヘッダーを統合
2. フィルターボタンをアイコンのみに簡素化
3. 月/週/日切替をより目立つ位置に配置

### Phase 2: カレンダービュー最適化
1. 月表示のセル高さを最適化
2. 週表示・日表示のレイアウト改善
3. 予定カードの表示を改善

### Phase 3: モーダル・フォーム改善
1. モーダルのフルスクリーン化
2. 予定詳細画面の情報充実
3. フォームレイアウトの最適化

### Phase 4: 細部の調整
1. FABとタップターゲットの最適化
2. 余白の統一
3. 視覚的階層の強化
