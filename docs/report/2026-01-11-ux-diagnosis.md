# UX診断レポート: コンセプト適合度評価

**診断日:** 2026-01-11
**対象:** AI Assistant Scheduler
**コンセプト:** 「予定を入れる時にGoogle検索するのが面倒」を解決する

---

## 総合評価

| 項目 | 評価 |
|------|------|
| コンセプト適合度 | ◎ 良好 |
| コア体験の完成度 | 高い |
| 改善余地 | あり（後述） |

---

## 検証したユーザーフロー

1. カレンダーの日付をクリック
2. 予定作成モーダルが開く
3. タイトルを入力（例: 「渋谷 イタリアン ディナー」）
4. 「AIで補完」ボタンをクリック
5. キーワード提案画面が表示（デート、おしゃれ、ワイン等）
6. キーワードを選択して「検索する」
7. AI検索結果が表示（店舗候補、条件判定、営業時間等）
8. 店舗選択カードから選択
9. 予定がカレンダーに自動保存

---

## うまくいっている点

### 1. 入力の簡便さ
- タイトル入力 → キーワード選択 → 検索、の**3ステップで完結**
- プレースホルダー（例: 都内 レストラン 新宿）で入力形式を示唆

### 2. 検索の自動化
- 複数サイト（ぐるなび、一休、Yahoo!マップ、食べログ等）を**横断して情報収集**
- ユーザーが各サイトを個別に検索する必要がない

### 3. 条件判定の可視化
- 必須条件: ✅ / ❌
- 優先条件: ✅ / △
- 重視ポイント: ✅ / —

各店舗が条件を満たしているか**一目でわかる**UI

### 4. 営業日確認
- 指定日（予定の日付）に営業しているかを**自動チェック**
- 定休日で行けない店舗を事前に除外

### 5. 結果の自動保存
- 検索結果が予定に紐づいて自動保存
- 後から「なぜこの店を選んだか」を振り返れる

### 6. こだわり条件
- ユーザープロファイルとして一度設定すれば、**全ての検索に自動適用**
- 3段階の条件（必須/優先/重視）で優先度を明確化

**現在の設定例:**
- 必須条件: 禁煙
- 優先条件: 駅近、個室あり
- 重視するポイント: 口コミ評価4.0以上

## 改善の余地がある点

### 1. 「検索しない予定」との棲み分け

**現状:**
- 「週次ミーティング」のような検索不要な予定も同じフォームを通る
- 「AIで補完」ボタンが目立つ位置にある

**検討案:**
- カテゴリ（仕事/プライベート）によってAIボタンの表示を変える
- または「検索が必要な予定」と「単純な予定」でフォームを分ける

### 2. 店舗選択後の体験

**現状:**
- 店舗を選択した後の導線が不明確

**検討案:**
- 選択した店舗の予約リンクへの導線
- 住所のコピー機能
- Googleマップへの連携

### 3. キーワードの自由入力

**現状:**
- AI提案のキーワードから選択する形式
- 「別のキーワードを提案してもらう」ボタンはある

**検討案:**
- 自由入力欄を追加（提案が的外れな場合の保険）

### 4. 検索時間の体感

**現状:**
- AI検索に数秒〜十数秒かかる

**検討案:**
- プログレス表示の改善（何を検索中か表示）
- ストリーミング表示で途中経過を見せる（現状でも部分的に実装済み）

## 「こだわり条件」と「予定詳細（メモ）」の棲み分け

### 現状の設計思想

| 機能 | スコープ | 用途 |
|------|----------|------|
| こだわり条件 | ユーザー全体 | 「私はいつもこれを重視する」 |
| 予定詳細（メモ） | その予定だけ | 「今回はこれがポイント」 |

### 懸念点

- 予定詳細で条件を入力できるようになると、こだわり条件との棲み分けが曖昧になる
- ただし、予定詳細では「禁止条件」のような強い制約を表現しにくい
- こだわり条件の3段階（必須/優先/重視）はその点で明確

### 結論

**現時点では両方維持**し、実際に使い込んでから判断する。
- 「やっぱりこだわり条件も毎回書き直してる」→ 統合を検討
- 「うまく棲み分けできてる」→ そのまま

---

## 総評

コア体験（タイトル → キーワード選択 → 検索結果 → 店舗選択）は**Google検索より明らかに楽**。

特に以下の自動化が価値を生んでいる:
- 営業日確認
- 条件判定（必須/優先/重視）
- 複数サイト横断検索
- 結果の構造化と保存

「予定を入れる時にGoogle検索するのが面倒」というペインに対して、**コンセプトに沿った設計ができている**。

## 次のアクション候補

1. 店舗選択後の体験を改善（予約導線、地図連携）
2. 検索不要な予定のフロー最適化
3. キーワード自由入力の追加
4. 使い込みながら「こだわり条件」の必要性を再評価
